## Code Quality Standards

### Validation Checks (CI)
pnpm typecheck    # No type errors
pnpm lint         # ESLint enforced
pnpm format       # Prettier applied
pnpm build        # Build succeeds
pnpm test         # All tests pass

### Additional Checks
- Docs - All links valid
- TypeDoc - Functions documented
- Comments - No inline `//`
- Prompts - XML structure required
- Security - No secrets in code

### TypeScript Requirements
- TypeScript strict mode
- No `any` types without justification
- Properly typed parameters & return values
- Use interfaces for object shapes
- TypeDoc comments on all exports

### Comment Style
- ‚ùå NO inline comments in production code
- ‚úÖ TypeDoc comments above functions
- ‚úÖ Self-documenting through clear naming

### Code Organization
- One primary export per file
- Keep files under 300 lines
- camelCase, PascalCase, SCREAMING_SNAKE_CASE
