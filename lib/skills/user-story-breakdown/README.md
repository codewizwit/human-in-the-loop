# User Story Breakdown

Breaks down epics and features into actionable, testable user stories following Agile best practices. Generates story point estimates, acceptance criteria, technical considerations, and identifies dependencies for effective sprint planning.

## What You'll Be Asked

- **Epic description**: What feature or epic needs to be broken down
- **User personas**: Who will use this feature (optional, can be inferred)
- **Technical context**: Tech stack, existing systems, constraints (optional, can be inferred from codebase)
- **Estimation method**: Fibonacci, T-Shirt sizes, linear, or none

## Usage Examples

- "Break down a user authentication system epic into stories"
- "Create user stories for a shopping cart feature"
- "Help me plan sprint stories for our API migration"
- "Break down this feature request into actionable Agile stories with acceptance criteria"
