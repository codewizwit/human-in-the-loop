# Test Coverage Analysis

Analyzes test coverage reports and codebase to identify gaps, assess risk by business impact, prioritize testing efforts, and generate actionable recommendations with specific test cases and effort estimates.

## What You'll Be Asked

- **Coverage report**: Istanbul/NYC JSON, lcov summary, or Jest output
- **Target coverage**: Minimum viable (70%), standard (80%), high (90%), or custom
- **Priority areas**: Specific modules or features to focus on (optional)
- **Codebase context**: Structure and critical paths description (optional)

## Usage Examples

- "Analyze our coverage report and tell me where to focus testing efforts"
- "We're at 67% coverage and need to reach 80% before release - create a plan"
- "Find the riskiest coverage gaps in our NestJS API"
- "Generate a prioritized action plan to improve test coverage for the payments module"
