# Pipeline Optimization

Analyzes CI/CD pipelines in your workspace for speed, cost, and reliability. Uses Read and Glob to find pipeline configs (GitHub Actions, GitLab CI, CircleCI, Jenkins). Identifies bottlenecks, suggests parallelization, caching, and provides estimated savings.

## What You'll Be Asked

- **CI platform**: GitHub Actions, GitLab CI, CircleCI, Jenkins, or auto-detect
- **Optimization goals**: Speed, cost, reliability, developer experience, or all
- **Current metrics**: Average duration, monthly cost, and success rate (if known)

## Usage Examples

- "Analyze our CI/CD pipeline and suggest optimizations to reduce duration and cost"
- "Review the pipeline configuration with focus on developer experience"
- "Our pipeline takes 12 minutes on average. Please analyze and optimize."
- "Help me add caching to our GitHub Actions workflow"
